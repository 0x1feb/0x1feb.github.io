SSH 接続で起動した Podman のコンテナを SSH 切断後も停止しないようにする

2025-02-06 · 1 min · Me | Suggest Changes
Table of Contents

概要#

  • Ubuntu 24.04 LTS に SSH 接続して Podman のコンテナを実行し、SSH 切断するとコンテナが停止してしまう。
  • SSH 切断後もコンテナが動作し続けるように対処する。

環境#

  • Ubuntu Server 24.04 LTS
  • podman version 4.9.3

手順#

Podman サービスの自動起動設定#

以下コマンドを実行し、サーバ起動時にコンテナが自動起動するようにする。

systemctl --user enable --now podman-restart

Linger 設定#

ユーザのログアウト後も systemctl --user を動かし続けるための設定をする。

以下コマンドを実行し、Linger の値を確認する。

loginctl show-user USERNAME | grep Linger

Linger=no

以下コマンドを実行し、Linger を有効化する。

sudo loginctl enable-linger USERNAME

Podman コンテナの開始#

任意の Podman コンテナを開始する。ここでは、以前作成した Open WebUI のコンテナを起動する。

podman start open-webui

SSH 切断しても Open WebUI のサービスに接続できることを確認する。
