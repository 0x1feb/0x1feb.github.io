Ubuntu に Syncthing をインストールして複数端末間のローカルファイルを同期する

概要#

  • Syncthing は、ローカルにあるファイルを複数端末間で同期できるソフトウェアである。
  • Ubuntu/Windows/Android 間で同じファイルを同期するよう設定する。

前提#

  • Syncthing v1.28.1

手順#

Syncthing のインストール#

各端末に Syncthing をインストールする。

Windows#

Windows は、以下よりインストーラをダウンロードし、インストールする。

  • Syncthing ｜ Downloads

Android#

Android は、以下より Syncthing-Fork をインストールする。

  • Syncthing-Fork - Google Play のアプリ

Ubuntu#

Ubuntu は、以下 URL を参考にインストールする。

  • Syncthing

トラブルシューティングに記載されているアプリを先にインストールしておく。

sudo apt-get install apt-transport-https ca-certificates

Syncthing の設定を CLI で確認する場合、JSON 形式のため、パースするための jq もインストールしておく。

sudo apt-get install jq

後は手順通り進める。

# Add the release PGP keys:
sudo mkdir -p /etc/apt/keyrings
sudo curl -L -o /etc/apt/keyrings/syncthing-archive-keyring.gpg https://syncthing.net/release-key.gpg

# Add the "stable" channel to your APT sources:
echo "deb [signed-by=/etc/apt/keyrings/syncthing-archive-keyring.gpg] https://apt.syncthing.net/ syncthing stable" | sudo tee /etc/apt/sources.list.d/syncthing.list

# Add the "candidate" channel to your APT sources:
echo "deb [signed-by=/etc/apt/keyrings/syncthing-archive-keyring.gpg] https://apt.syncthing.net/ syncthing candidate" | sudo tee /etc/apt/sources.list.d/syncthing.list

# Update and install syncthing:
sudo apt-get update
sudo apt-get install syncthing

syncthing v1.28.1-rc.1 "Gold Grasshopper" (go1.23.3 linux-amd64) debian@github.syncthing.net 2024-11-12 09:50:19 UTC [noupgrade]

Syncthing の設定#

Ubuntu <=> Windows 間と Ubuntu <=> Android 間で、Obsidian の vault を同期するように設定する。

ID の生成#

各端末でデバイス ID を確認する。Ubuntu では、以下のコマンドを実行する。

syncthing generate

2024/11/14 23:27:06 INFO: Generating ECDSA key and certificate for syncthing...
2024/11/14 23:27:06 INFO: Device ID: UBUNTUX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX
2024/11/14 23:27:06 INFO: Default folder created and/or linked to new config

今回は、端末 ID を以下として説明する。

  • Ubuntu: UBUNTUX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX
  • Windows: WINDOWS-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX
  • Android: ANDROID-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX

サービスの開始#

以下コマンドにて、Syncthing が起動する。今回はコマンドラインで設定するため、--no-browser オプションを付ける。

syncthing serve --no-browser

サービスとして登録する場合、以下コマンドを実行する。{user} に設定した Syncthing のユーザアカウントを作成し、サーバ起動時に Syncthing が自動で実行されるようになる。

systemctl enable syncthing@{user}.service
systemctl start syncthing@{user}.service

端末の設定#

Ubuntu にて、対象端末のデバイス ID を設定する。デバイス ID は、Windows や Android の Syncthing より確認する。

syncthing cli config devices add --device-id "WINDOWS-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX"

上記を実行すると、追加される側である Windows の管理画面にて、対象に追加するかを選ぶダイアログが表示される。Android の端末 ID も同様に追加する。

同期対象の設定#

以下コマンドにて、/data/vault にあるディレクトリを vault というラベルで同期するための Syncthing ディレクトリを作成する。obsidian-vault は、端末間で共有する識別 ID である。

syncthing cli config folders add --id "obsidian-vault" --label "vault" --path "/data/vault"

以下コマンドにて、同期対象として指定可能なディレクトリ設定一覧を出力し、obsidian-vault が作成されていることを確認する。

syncthing cli config folders list

default
obsidian-vault

以下コマンドを実行し、設定したディレクトリパスを確認する。 (以下は obsidian-vault のパス)

syncthing cli config folders obsidian-vault dump-json | jq .path

"/data/vault"

obsidian-vault を指定したデバイスと共有する。以下コマンドは、Windows 端末に対して obsidian-vault を共有するコマンドである。

syncthing cli config folders "obsidian-vault" devices add --device-id "WINDOWS-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX-XXXXXXX"

上記コマンドを実行すると、Windows 側の管理画面にて、対象に追加するかのダイアログが表示される。

Android の端末 ID に対しても同様に共有ディレクトリを追加する。

確認#

これで Windows と Android のローカルにある Obsidian のファイルが、Ubuntu と同期されるようになる。

Ubuntu は自宅サーバとして使用しているため、共有フォルダのように使うことができる。

Ubuntu がダウンしているときは、Windows と Android のローカルにあるファイルは同期されず、Ubuntu が復旧したタイミングで同期される。

Windows と Android 間も同期すると、Ubuntu がダウンしても同期できるようになる。
