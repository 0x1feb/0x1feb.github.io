PowerShell で特定ポートの疎通を確認する

概要#

  1. PowerShell の Test-NetConnection を使用する。
  2. .NET の System.Net.Sockets.TcpClient を使用する。

前提#

  • PowerShell 5.1/7.4

手順#

Test-NetConnection#

Test-NetConnection を使う方法を説明する。

例えば、192.168.1.1 の 22 ポートを確認する場合、PowerShell にて、以下のコマンドを実行する。

Test-NetConnection 192.168.1.1 -Port 22

成功すると、次の結果が得られる。

ComputerName     : 192.168.1.1
RemoteAddress    : 192.168.1.1
RemotePort       : 22
InterfaceAlias   : イーサネット
SourceAddress    : 192.168.11.13
TcpTestSucceeded : True

PowerShell スクリプト等で結果を使って処理する場合、以下のように実行結果オブジェクトを受け取る。

$result = Test-NetConnection 192.168.1.1 -Port 22
if ($result.TcpTestSucceeded) {
  Write-Host "$($result.ComputerName) listening to port $($result.RemotePort)"
}

成功した場合、if 文の処理が実行される。

192.168.1.1 listening to port 22

TcpClient#

TcpClient を使う方法を説明する。

例えば、192.168.1.1 の 22 ポートを確認する場合、PowerShell にて、以下のコマンドを実行する。

New-Object System.Net.Sockets.TcpClient("192.168.1.1", 22)

成功すると、次の結果が得られる。

Client              : System.Net.Sockets.Socket
Available           : 0
Connected           : True
ExclusiveAddressUse : False
ReceiveBufferSize   : 65536
SendBufferSize      : 65536
ReceiveTimeout      : 0
SendTimeout         : 0
LingerState         : System.Net.Sockets.LingerOption
NoDelay             : False

PowerShell スクリプト等で結果を使って処理する場合、以下のように実行結果オブジェクトを受け取る。

$ip = "192.168.1.1"
$port = 22
$result = New-Object System.Net.Sockets.TcpClient($ip, $port)
if ($result.Connected) {
  Write-Host "$ip listening to port $port"
}

成功した場合、if 文の処理が実行される。

192.168.1.1 listening to port 22
