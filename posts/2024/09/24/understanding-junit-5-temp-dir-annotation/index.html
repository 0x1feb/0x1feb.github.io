JUnit 5 の @TempDir でテスト実行時のみの一時ディレクトリを作成する

2024-09-24 · 2 min · Me | Suggest Changes
Table of Contents
  • 概要
  • 前提
  • 説明
    • @TempDir
  • 参考

概要#

  • JUnit 5 のテストにて、テスト実行時のみに使用する一時ディレクトリを作成する。
    • @TempDir
  • テスト実行時のみ実態のあるファイルやディレクトリを扱いたいときに役立つ。

前提#

  • JUnit 5

説明#

@TempDir#

@TempDir は、一時ディレクトリを作成するアノテーションである。

@TempDir アノテーションを付与して一時ディレクトリが作成されるのは、以下のいずれかである。

  • テストクラスのフィールド
  • ライフサイクルメソッド
  • テストメソッドのパラメータ

一時ディレクトリの型は、以下のいずれかを指定する。

  • java.nio.file.Path
  • java.io.File

テストメソッドのパラメータに @TempDir を使用した例を示す。

class ExampleTempDirTest {
    @Test
    void tempDirTest(@TempDir Path tempDirPath) throws IOException {
        Path exampleFilePath = tempDirPath.resolve("example.txt");
        final String message = "hello, world!";
        Files.writeString(exampleFilePath, message);

        final String absolutePath = exampleFilePath.toAbsolutePath().toString();
        System.out.println(absolutePath);

        final String actual = Files.readString(Path.of(absolutePath));
        assertEquals(message, actual);
    }
}

テストメソッドのパラメータ @TempDir Path tempDirPath にて、一時ディレクトリが作成され、tempDirPath に一時ディレクトリへのパスが入る。

Path exampleFilePath = tempDirPath.resolve("example.txt");
final String message = "hello, world!";
Files.writeString(exampleFilePath, message);

テストディレクトリ配下に example.txt ファイルを作成し、hello, world! というテキストを書き込んでいる。

final String absolutePath = exampleFilePath.toAbsolutePath().toString();
System.out.println(absolutePath);

作成したファイルの絶対パスを取得し、コンソールに出力している。出力結果は以下の通りである。(junit2513515903013282556 の部分はテスト実行毎に変わる。)

  • C:\Users\xxx\AppData\Local\Temp\junit2513515903013282556\example.txt
final String actual = Files.readString(Path.of(absolutePath));
assertEquals(message, actual);

最後に、このパスのファイルから文字列を読み取り、hello, world! と一致することを確認している。

テストが終わると、上記のディレクトリは再帰的に削除される。削除に失敗した場合は、IOException が発生し、テストは失敗する。

以下のように、フィールドやライフサイクルメソッドに使うこともできる。

class ExampleTempDirTest {
    @TempDir
    private Path tempFilePath;

    @BeforeEach
    void init(@TempDir File initDir) {
      // ...
    }

    ...
}

この場合、一時ディレクトリは、ライフサイクルと同じタイミングで生成や削除される。
