JUnit 5 の動的テスト @TestTemplate を整理する

概要#

  • JUnit 5 の動的テストを整理する。
    • @TestTemplate

前提#

  • JUnit 5

解説#

@TestTemplate#

@TestTemplate を付与したメソッドは、登録したプロバイダから返されるコンテキストの数だけ実行されるテストメソッドとなる。プロバイダを個別に実装することで、テストメソッドで使用するコンテキストを柔軟に制御できる。

TestTemplateInvocationContextProvider#

まず、TestTemplateInvocationContextProvider インタフェースを実装したプロバイダクラスを作成する。

class ExampleTestTemplateProvider implements TestTemplateInvocationContextProvider {

    @Override
    public boolean supportsTestTemplate(ExtensionContext extensionContext) {
        return true;
    }

    @Override
    public Stream<TestTemplateInvocationContext> provideTestTemplateInvocationContexts(ExtensionContext extensionContext) {
        return Stream.of(invocationContext(3), invocationContext(1), invocationContext(4));
    }

    private TestTemplateInvocationContext invocationContext(Integer num) {
        return new TestTemplateInvocationContext() {
            @Override
            public String getDisplayName(int invocationIndex) {
                return "Template Test: index=%d, num=%d".formatted(invocationIndex, num);
            }

            @Override
            public List<Extension> getAdditionalExtensions() {
                return Collections.singletonList(new ParameterResolver() {
                    @Override
                    public boolean supportsParameter(ParameterContext parameterContext,
                                                     ExtensionContext extensionContext) {
                        return parameterContext.getParameter().getType().equals(Integer.class);
                    }

                    @Override
                    public Object resolveParameter(ParameterContext parameterContext,
                                                   ExtensionContext extensionContext) {
                        return num;
                    }
                });
            }
        };
    }
}

TestTemplateInvocationContextProvider の supportsTestTemplate は、このプロバイダがテストテンプレートメソッドの呼び出しコンテキストをサポートするかを判定する。false のときにテストテンプレートメソッドのテストを呼び出した場合、テストは失敗する。

@Override
public boolean supportsTestTemplate(ExtensionContext extensionContext) {
    return true;
}

TestTemplateInvocationContextProvider の provideTestTemplateInvocationContexts は、テストテンプレートメソッドの呼び出しコンテキストを設定する。テストに提供したい TestTemplateInvocationContext の Stream を返す。

@Override
public Stream<TestTemplateInvocationContext> provideTestTemplateInvocationContexts(ExtensionContext extensionContext) {
    return Stream.of(invocationContext(3), invocationContext(1), invocationContext(4));
}

ここでは、TestTemplateInvocationContext を生成する invocationContext メソッドを定義している。引数の Integer をもとに TestTemplateInvocationContext オブジェクトを生成する。

private TestTemplateInvocationContext invocationContext(Integer num) {
    return new TestTemplateInvocationContext() {
        ...
    };
}

TestTemplateInvocationContext の getDisplayName は、テストの表示名である。ここでは、インデックスと invocationContext の引数として受け取った num を使用し、表示名を動的に作成している。

@Override
public String getDisplayName(int invocationIndex) {
    return "Template Test: index=%d, num=%d".formatted(invocationIndex, num);
}

TestTemplateInvocationContext の getAdditionalExtensions は、Extension のリストを返す。今回は、ParameterResolver のシングルトンのリストを生成している。

@Override
public List<Extension> getAdditionalExtensions() {
    return Collections.singletonList(new ParameterResolver() {
        ...
    });
}

ParameterResolver の supportsParameter は、パラメータコンテキスト内のパラメータを検証する。ここでは、Integer 型であることを検証している。

@Override
public boolean supportsParameter(ParameterContext parameterContext,
                                 ExtensionContext extensionContext) {
        return parameterContext.getParameter().getType().equals(Integer.class);
}

ParameterResolver の resolveParameter は、パラメータコンテキストのパラメータを解決する。ここでは、invocationContext の引数として受け取った Integer 型の num をそのまま返している。前述した supportsParameter の結果が false になると、テストは失敗する。

@Override
public Object resolveParameter(ParameterContext parameterContext,
                               ExtensionContext extensionContext) {
    return num;
}

Test Class#

@TestTemplate を使用したテストクラスを作成する。

@ExtendWith(ExampleTestTemplateProvider.class)
class TemplateTest {

    @TestTemplate
    void testTemplate1(Integer number) {
        Assertions.assertTrue(number > 0, "Number should be greater than 0");
    }

    @TestTemplate
    void testTemplate2(Integer number) {
        Assertions.assertTrue(number < 10, "Number should be less than 10");
    }
}

テンプレートとして使用するプロバイダは、@ExtendWith で指定する。@ExtendWith は、プロバイダを適用したいテストクラスまたはテストメソッドに付与する。

テストメソッドに @TestTemplate を付与する。

テストの実行結果は次の通りである。

TemplateTest,testTemplate1 and 1 more: 6 total, 6 passed

TemplateTest
  testTemplate1(Integer)
    Template Test: index=1, num=3
    Template Test: index=2, num=1
    Template Test: index=3, num=4
  testTemplate2(Integer)
    Template Test: index=1, num=3
    Template Test: index=2, num=1
    Template Test: index=3, num=4

ExampleTestTemplateProvider で実装したテストが実行されていることがわかる。
