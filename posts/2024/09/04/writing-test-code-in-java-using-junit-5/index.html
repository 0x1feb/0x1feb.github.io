概要#

  • JUnit 5 について、最低限の動きを整理する。
    • @Test アノテーション
    • assertEquals
    • fail

前提#

  • JUnit 5
  • Java 8 (or higher)
    • 今回のサンプルは Java 21 で作成

手順#

ライブラリの設定#

JUnit 5 User Guide の以下の章を参照し、環境に合わせて JUnit 5 に必要なライブラリを設定する。

  • 4.1. IDE Support | JUnit 5 User Guide
  • 4.2. Build Support | JUnit 5 User Guide

テスト対象クラスの作成#

以下の Main クラスをテスト対象クラスとする。

public class Main {
    public Main() {
    }

    public String hello(String name) {
        return "Hello, %s.".formatted(name);
    }

    public String bye(String name) {
        return "Bye, %s.".formatted(name);
    }
}

各メソッドは、引数の String を使って、メッセージを作成し、作成したメッセージを返却する。

例えば、hello メソッドに文字列 "John Smith" を渡すと、文字列 "Hello, John Smith." を返却する。

テストクラスの作成#

Main クラスに対するテストとして、MainTest クラスを作成する。

大体の IDE には、Main クラスをもとに自動で MainTest を作成する機能がある。

class MainTest {
    @Test
    void testHello() {
    }

    @Test
    void testBye() {
    }
}

@Test アノテーションが付与されたメソッドは、テストメソッドとして扱われる。

以下の状態でテストを実行すると、2 件中 2 件のテストがパスする。

class MainTest {
    @Test
    void testHello() {
    }

    @Test
    void testBye() {
    }
}

テストメソッドは、何もしなくても成功扱いになる。後述するが、未実装のテストコードは失敗するようにしておいた方が良い。

テストメソッドの実装#

今回は hello メソッドのテストメソッド testHello を実装する。

完成した testHello は、以下の通りである。

@Test
void testHello() {
    // setup
    Main main = new Main();
    final String name = "John Smith";

    // exercise
    final String actual = main.hello(name);

    // verify
    final String expected = "Hello, John Smith.";
    assertEquals(expected, actual);
}

テストコードは一度書いたら終わりというものではなく、次のような対応でメンテナンスが必要となる。

  • テスト対象コードの変更に伴い、テストコードを修正する。
  • あるテストコードと別のテストコードの共通処理をまとめて整理する。

そのため、テストコードも可読性や保守性を意識して実装する。

今回は、テストメソッドを以下の 3 段階に分けて記載する。

  1. setup: テスト対象コード実行前の前処理
  2. exercise: テスト対象コード実行処理
  3. verify: テスト対象コード実行結果の検証処理

setup#

setup では、テスト対象コードを実行する前の準備処理を書く。

// setup
Main main = new Main();
final String name = "John Smith";

テスト対象クラスである Main クラスのインスタンスを作成している。また、hello メソッドの引数に渡す文字列 name を定義している。

exercise#

exercise では、テスト対象メソッドの実行処理を書く。

// exercise
final String actual = main.hello(name);

テスト対象クラスのインスタンスである main の hello メソッドを実行している。このとき、setup で準備した name を引数として渡している。

実行結果は、actual に格納する。

verify#

verify では、テスト対象コードの実行結果を検証する処理を書く。

// verify
final String expected = "Hello, John Smith.";
assertEquals(expected, actual);

hello メソッドは String を返すため、期待する String と 実際の戻り値である String が一致すれば、正しく実装できていることを検証できる。

期待する結果を expected に格納し、assertEquals メソッドで、期待する結果 expected と実際の結果 actual が等しいか検証する。

未実装のテストコード#

テストメソッド testHello は実装したが、testBye は未実装である。

@Test
void testBye() {
}

しかし、上記のコードだと、実行してもテスト自体はパスしてしまう。そのため、実装忘れを防ぐために、以下のコードを実装しておく。

@Test
void testBye() {
    fail("Not yet implemented");
}

fail メソッドは、実行されるとテストが失敗したことになる。

テストの実行#

今回は、テストメソッド testHello を実装し、testBye は未実装として常に失敗するようにした。

class MainTest {
    @Test
    void testHello() {
        // setup
        Main test = new Main();
        final String name = "John Smith";

        // exercise
        final String actual = test.hello(name);

        // verify
        final String expected = "Hello, John Smith.";
        assertEquals(expected, actual);
    }

    @Test
    void testBye() {
        fail("Not yet implemented");
    }
}

テストは、クラス単位やメソッド単位でも実行できる。

今回の MainTest を実行すると、次の結果になる。

  • testHello: pass
  • testBye: fail (Not yet implemented)

testHello が失敗している場合、hello メソッドまたは testHello メソッドに誤りがある。テストの実行結果に詳細が出力されるため、その内容をもとに修正する。
