概要#

  • bash のシェルスクリプトでログ出力する方法を色々と調べたので整理した。

$ bash --version

ログ出力方法#

リダイレクトする#

#!/bin/bash

# 標準出力をログにリダイレクトする
echo "standard output" 1>output.log
# 省略した場合も同じ
echo "standard output (implicit)" >output.log

# 標準エラー出力をログにリダイレクトする
ls --unrecognized-option 2>output.log

# 追記する
echo >> output.log

# 標準エラー出力を標準出力にリダイレクトする
ls --unrecognized-option >output.log 2>&1
# 標準出力と標準エラー出力をログにリダイレクトする
ls --unrecognized-option &>output.log

ちょっとしたスクリプトならこれで問題ないが、ある程度大きくなると、全てに付与するのが面倒である。

{…} で括って一括出力する#

#!/bin/bash

{
  echo "hello, world!"
  ls --unrecognized-option
} &>output.log

{...} の中で実行されたコマンドの標準出力と標準エラー出力をログに出力する。

ある程度まとまった処理ごとにログを分けたい場合に使えそう。

exec コマンドを使って一括設定する#

#!/bin/bash

exec &>output.log

echo "hello, world!"
ls --unrecognized-option

標準出力と標準エラー出力をログファイルにリダイレクトする。

実行すると、output.log は以下のような中身になる。

hello, world!
ls: unrecognized option '--unrecognized-option'
Try 'ls --help' for more information.

いちいちリダイレクト処理を書かなくてよいので楽になる。

コンソールにも出力する#

#!/bin/bash

exec &> >(tee -a output.log)

echo "hello, world!"
ls --unrecognized-option

>(command) はプロセス置換という。

標準出力と標準エラー出力を tee コマンドに渡して、コンソールとログに出力している。

各行の先頭に日付(yyyy-MM-dd hh:mm:ss)を付与する#

#!/bin/bash

exec &> >(awk '{print strftime("%Y-%m-%d %H:%M:%S ") $0; fflush();}')

echo "hello, world!"
ls --unrecognized-option

awk コマンドを使って、日付 strftime("%Y-%m-%d %H:%M:%S ") を付与している。

$0 で標準出力と標準エラー出力の全量が渡される。

fflush() でバッファせずに出力する。

実行すると、次のようなログがコンソールに出力される。

$ bash main.sh
2024-03-13 23:08:39 hello, world!
2024-03-13 23:08:39 ls: unrecognized option '--unrecognized-option'
2024-03-13 23:08:39 Try 'ls --help' for more information.

コンソールとログの両方に出力したい場合は、awk コマンドの出力を tee コマンドに渡す。

#!/bin/bash

exec &> >(awk '{print strftime("%Y-%m-%d %H:%M:%S ") $0; fflush();}' | tee -a "$LOG_FILE")
