PowerShell のコマンドでプロセスを開始する

概要#

  • PowerShell の Start-Process コマンドでプロセスを開始する。

環境#

  • PowerShell 7.4.5

説明#

新たな PowerShell プロセス (pwsh.exe) を開始する例で説明する。他のアプリやファイルを実行する場合は、pwsh.exe 部分を置き換える。

通常の呼び出し (非同期)#

-FilePath で実行するプロセス (今回は PowerShell である pwsh.exe) を指定する。

Start-Process -FilePath pwsh.exe

上記を実行すると、新しい PowerShell を開く。

デフォルトは非同期実行となる。そのため、後続処理がある場合、開始したプロセスの終了を待たずに次の処理が実行される。例えば、以下を実行すると、新しい PowerShell を開いた直後に Write-Host を実行する。

Start-Process -FilePath pwsh.exe
Write-Host "PowerShell プロセスを開始しました。"

また、呼び出し側の環境変数は引き継がれる。例えば、以下のように呼び出し元で環境変数を定義してから実行する。

$env:EXAMPLE_VALUE = "example value"
Start-Process -FilePath pwsh.exe

新たに開始した PowerShell で環境変数の値を確認すると、設定値を引き継いでいることがわかる。

> Write-Host $env:EXAMPLE_VALUE
example value

同期#

プロセスの終了後に後続処理を実行したい場合、-Wait を指定する。

Start-Process -FilePath pwsh.exe -Wait

上記を実行すると、新しい PowerShell を開く。新しく開いた PowerShell を閉じると、後続処理が実行される。

引数を指定して起動#

引数を指定して呼び出す場合、-ArgumentList を指定する。

例えば、PowerShell (pwsh.exe) は、-Command を指定すると、呼び出しと同時にコマンドを実行できる。この場合、コマンドを実行するとプロセスは終了してしまうが、-NoExit を指定すると、ウィンドウが開いたままになる。

Start-Process -FilePath pwsh.exe -ArgumentList "-NoExit", "-Command", "Get-Process"

上記を実行すると、pwsh.exe -NoExit -Command Get-Process と同じように起動できる。

管理者として起動#

新しいプロセスを管理者として実行したい場合、-Verb RunAs を指定する。

Start-Process -FilePath pwsh.exe -Verb RunAs

上記を実行すると、管理者権限で新しい PowerShell を開く。

ワーキングディレクトリを指定して起動#

ワーキングディレクトリを指定する場合、-WorkingDirectory を指定する。

Start-Process -FilePath pwsh.exe -WorkingDirectory "D:/tmp"

上記を実行すると、カレントディレクトリを D:\tmp とした状態で PowerShell を開く。

終了コードの取得#

プロセスの終了コードを取得したい場合、-PassThru を指定する。-PassThru を指定すると、Start-Process は実行したプロセスのプロセスオブジェクトを返す。

$process = Start-Process -FilePath "pwsh.exe" -PassThru -Wait

switch ($process.ExitCode) {
    0 { Write-Host "Success (Exit Code: $($process.ExitCode))" }
    1 { Write-Host "Warning (Exit Code: $($process.ExitCode))" }
    default { Write-Host "Error (Exit Code: $($process.ExitCode))" }
}

上記を実行すると、起動した PowerShell プロセスの終了コードによってメッセージを出力する。例えば、起動した PowerShell で Exit 0 を実行すると、呼び出し元のコンソールに Success (Exit Code: 0) を出力する。
